# 区域配置和编辑功能改进说明

## 改进内容

### 1. 修复 PUT /admin/credentials/:id 鉴权问题

**问题**: 前端编辑凭据时调用 `PUT /admin/credentials/:id` 返回 405 错误

**原因**: 前端使用的 `useUpdateCredential` hook 直接使用原生 `fetch` 调用，没有包含鉴权 header（x-api-key 的 MD5 哈希）

**修复内容**:
- 在 `src/api/credentials.ts` 中新增 `updateCredential()` 函数，使用带鉴权拦截器的 axios 实例
- 更新 `useUpdateCredential` hook 使用新的 API 函数，自动附加 MD5 哈希的 x-api-key header

**修改文件**:
```
admin-ui/src/api/credentials.ts       - 新增 updateCredential() 函数
admin-ui/src/hooks/use-credentials.ts - 更新 hook 使用新函数并导入
```

**鉴权机制**:
```typescript
// axios 拦截器自动添加 MD5 哈希的 API Key
api.interceptors.request.use((config) => {
  const apiKey = storage.getApiKey()
  if (apiKey) {
    const hashedKey = md5(apiKey)
    config.headers['x-api-key'] = hashedKey
  }
  return config
})
```

### 2. 前端：一键获取区域功能

**需求**: 从 AWS SSO Portal URL 自动提取区域配置

**实现位置**: 编辑凭据对话框（`EditCredentialDialog`）

**功能说明**:
1. 点击"一键获取区域"按钮
2. 弹出输入框，粘贴 AWS SSO Portal URL
3. 自动提取区域信息，填入 Region、Auth Region、API Region 三个字段

**支持的 URL 格式**:
```
https://{instance-id}.portal.{region}.app.aws/
```

**示例**:
```
输入: https://ssoins-69874d8e815c8147.portal.eu-central-1.app.aws/
提取: eu-central-1
结果: 自动填入 region、authRegion、apiRegion 为 "eu-central-1"
```

**常见区域**:
- `us-east-1` - 美国东部（弗吉尼亚）
- `us-west-2` - 美国西部（俄勒冈）
- `eu-central-1` - 欧洲（法兰克福）
- `eu-west-1` - 欧洲（爱尔兰）
- `ap-southeast-1` - 亚太（新加坡）
- `ap-northeast-1` - 亚太（东京）

**修改文件**:
```
admin-ui/src/components/edit-credential-dialog.tsx
```

**实现细节**:
- 使用正则表达式 `\.portal\.([a-z0-9-]+)\.app\.aws` 提取区域
- 提取成功后同时填入三个区域字段
- 提取失败时显示友好错误提示

### 3. 前端：新增 API Key 凭据默认端点为 cli

**需求**: 新增 API Key 类型凭据时，默认设置端点为 `cli`

**实现位置**: 添加凭据对话框（`AddCredentialDialog`）

**功能说明**:
1. 当认证方式选择为 "API Key" 时
2. 自动设置 Endpoint 字段为 `"cli"`
3. 用户可以手动修改为其他值

**实现方式**:
```typescript
const handleAuthMethodChange = (method: AuthMethod) => {
  setAuthMethod(method)
  if (method === 'api_key' && !endpoint) {
    setEndpoint('cli')
  }
}
```

**修改文件**:

## 使用场景

### 场景 1: 编辑现有 API Key 凭据配置区域

1. 在凭据列表中点击"编辑"按钮
2. 点击"一键获取区域"按钮
3. 输入 AWS SSO Portal URL（如 `https://xxx.portal.eu-central-1.app.aws/`）
4. 点击"提取区域"
5. 系统自动填入 `eu-central-1` 到三个区域字段
6. 点击"保存"更新配置（自动附加 MD5 哈希的鉴权 header）

### 场景 2: 新增 API Key 凭据

1. 点击"添加凭据"
2. 认证方式选择 "API Key"
3. **Endpoint 字段自动填入 "cli"**
4. 输入 Kiro API Key
5. 配置区域（可使用一键获取功能）
6. 保存

## 技术实现

### 鉴权流程

**问题**: 原 `useUpdateCredential` 使用原生 fetch，没有 x-api-key header

**解决方案**: 使用统一的 axios 实例，拦截器自动添加 MD5 哈希

```typescript
// API 层（src/api/credentials.ts）
const api = axios.create({
  baseURL: '/api/admin',
})

api.interceptors.request.use((config) => {
  const apiKey = storage.getApiKey()
  if (apiKey) {
    config.headers['x-api-key'] = md5(apiKey)
  }
  return config
})

export async function updateCredential(
  updateData: Record<string, unknown>
): Promise<SuccessResponse> {
  const id = updateData.id
  const { data } = await api.put<SuccessResponse>(`/credentials/${id}`, updateData)
  return data
}

// Hook 层（src/hooks/use-credentials.ts）
export function useUpdateCredential() {
  return useMutation({
    mutationFn: updateCredential,  // 自动使用带鉴权的 axios
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['credentials'] })
    },
  })
}
```

### URL 区域提取算法

```typescript
const extractRegionFromSSO = (url: string): string | null => {
  // 匹配格式：https://{instance-id}.portal.{region}.app.aws/
  const match = url.match(/\.portal\.([a-z0-9-]+)\.app\.aws/i)
  return match ? match[1] : null
}
```

**匹配逻辑**:
- `.portal.` - 固定前缀
- `([a-z0-9-]+)` - 捕获区域名称（字母、数字、连字符）
- `.app.aws` - 固定后缀
- `i` - 不区分大小写

**支持的 URL 变体**:
```
✅ https://xxx.portal.eu-central-1.app.aws/
✅ https://xxx.portal.us-east-1.app.aws/login
✅ https://xxx.PORTAL.ap-northeast-1.APP.AWS/
❌ http://xxx.portal.invalid-region.com/
❌ https://xxx.eu-central-1.amazonaws.com/
```
