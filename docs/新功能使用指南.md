# 新功能使用指南

## 概述

本次更新新增了智能账号过滤和模型管理功能，提升了多账号场景下的请求成功率和可观测性。

## 智能账号过滤

### 功能说明

当你发送 API 请求时，系统会自动识别请求中的模型参数（如 `claude-opus-4.5`），并从模型服务中查询支持该模型的账号列表。只有支持该模型的账号才会被选中处理请求。

### 工作原理

1. **请求解析**: 从请求体中提取模型名称
2. **缓存查询**: 检查模型→账号映射缓存（默认 5 分钟 TTL）
3. **过滤账号**: 根据缓存结果过滤出支持该模型的账号
4. **负载均衡**: 在过滤后的账号中应用负载均衡策略（priority/balanced）
5. **回退机制**: 如果模型服务无数据，回退到基于订阅等级的 Opus 过滤

### 配置

模型刷新配置（可选，在 `config.json` 中）：

```json
{
  "model_refresh": {
    "enabled": true,
    "interval_seconds": 3600,
    "account_filter_cache_ttl_seconds": 300
  }
}
```

- `enabled`: 是否启用后台自动刷新（默认 false）
- `interval_seconds`: 刷新间隔，单位秒（默认 3600）
- `account_filter_cache_ttl_seconds`: 过滤缓存有效期，单位秒（默认 300）

### 日志

启用 DEBUG 日志可以看到过滤过程：

```bash
RUST_LOG=debug ./kiro-rs
```

示例日志输出：

```
[DEBUG] Model claude-opus-4.5 is supported by 2 accounts: [1, 3]
[INFO] 选择凭据 #1 处理请求
```

## 模型管理

### 访问模型管理界面

1. 访问 Admin UI：`http://127.0.0.1:8990/admin`
2. 输入你的 `adminApiKey` 登录
3. 点击顶部的 "Models" 标签页

### 模型列表功能

在模型列表中，你可以看到：

- **模型基本信息**: 名称、ID、类型、所有者
- **Token 限制**: 最大输出 Token 数、上下文窗口大小
- **能力标识**: Vision 支持、输入模态性（text/image）
- **定价信息**: 输入和输出 Token 的价格（如果有）
- **支持账号**: 哪些账号支持该模型，显示账号 ID

### 刷新模型列表

#### 刷新所有账号

点击右上角的 "Refresh All" 按钮：

- 系统会依次刷新所有启用的账号
- 显示进度提示：成功 X 个，失败 Y 个
- 刷新完成后自动重新加载模型列表

#### 刷新单个账号（API）

```bash
curl -X POST http://127.0.0.1:8990/api/admin/models/refresh/1 \
  -H "x-api-key: your-admin-api-key"
```

响应示例：

```json
{
  "success": true,
  "message": "账号 1 刷新成功，获取 15 个模型",
  "account_id": 1,
  "model_count": 15
}
```

### 首次启动

服务启动时会自动预加载所有账号的模型列表：

```
[INFO] 正在预加载模型列表...
[INFO] 模型预加载完成: 成功 3 个账号, 失败 0 个账号
```

如果所有账号刷新失败，服务仍会启动，但模型列表为空，账号过滤功能会回退到基于订阅等级的过滤。

### 后台自动刷新

如果启用了 `model_refresh.enabled: true`，系统会在后台定时刷新模型列表：

```
[INFO] 后台模型刷新任务已启动，间隔: 3600 秒
[DEBUG] 执行定时模型刷新
[INFO] 定时刷新完成: 成功 3 个账号, 失败 0 个账号
```

## 错误处理改进

### 账号刷新失败

当某个账号的 Token 刷新失败时：

1. **临时失败**: 累计到 `refresh_failure_count`，达到阈值（3次）后禁用该账号
2. **永久失效**: 立即禁用该账号，标记禁用原因为 `RefreshTokenInvalid`
3. **自动切换**: 立即尝试下一个可用账号
4. **自愈机制**: 所有账号都被禁用时，重置失败计数并重新启用

### 请求失败

API 请求失败时的智能处理：

| 状态码 | 处理策略 |
|--------|---------|
| 401/403 | Token 疑似失效 → 尝试强制刷新（每账号一次） → 刷新失败则切换账号 |
| 402 | 额度用尽 → 立即禁用该账号 → 切换到下一个 |
| 400 | 客户端请求错误 → 不重试，直接返回错误 |
| 429/408/5xx | 上游瞬态错误 → 重试但不禁用账号（避免误禁） |

### 重试策略

- 单个账号最多重试 3 次
- 单个请求最多重试 9 次（跨账号）
- 指数退避策略：200ms → 400ms → 800ms（最大 2s）

## 最佳实践

### 1. 定期刷新模型列表

建议启用后台自动刷新，或至少每天手动刷新一次，以确保模型信息是最新的。

### 2. 监控账号状态

定期查看 Admin UI 中的凭据状态：

- 检查是否有账号被自动禁用
- 查看失败计数和成功计数
- 确认账号余额充足

### 3. 合理配置缓存 TTL

- 账号过滤缓存 TTL（默认 300 秒）: 
  - 太短: 频繁计算，影响性能
  - 太长: 账号状态变化响应慢
  - 推荐: 300-600 秒

- 模型刷新间隔（默认 3600 秒）:
  - 太短: 频繁刷新，增加 API 调用
  - 太长: 新模型发现慢
  - 推荐: 3600-7200 秒

### 4. 日志监控

关注以下日志关键词：

- `Model X is supported by Y accounts`: 账号过滤成功
- `No accounts found for model X`: 无账号支持该模型
- `using fallback filter`: 回退到旧的过滤逻辑
- `refreshToken 永久失效`: 账号凭据过期，需要更新
- `额度已用尽`: 账号余额不足

## 故障排查

### 问题：所有请求都失败，提示"所有凭据均已禁用"

**原因**: 所有账号的 Token 刷新失败或被禁用

**解决方法**:

1. 查看日志，找到失败原因
2. 在 Admin UI 中点击"恢复异常"按钮重置失败计数
3. 或手动更新 `credentials.json` 中的凭据
4. 重启服务

### 问题：模型列表为空

**原因**: 账号刷新失败或未启用模型管理

**解决方法**:

1. 在 Admin UI 点击"Refresh All"手动刷新
2. 查看日志，确认刷新失败原因
3. 确认所有账号的凭据有效
4. 检查网络连接和代理配置

### 问题：某些模型请求总是失败

**原因**: 
- 账号不支持该模型
- 模型名称不正确
- 免费账号请求 Opus 模型

**解决方法**:

1. 在 Models 页面查看哪些账号支持该模型
2. 确认请求中的模型名称正确
3. 为免费账号请求 Sonnet 或 Haiku 模型

## API 参考

### GET /api/admin/models

获取所有模型列表。

**响应示例**:

```json
{
  "models": [
    {
      "id": "claude-sonnet-4-20250514",
      "object": "model",
      "created": 1744675200,
      "owned_by": "anthropic",
      "display_name": "Claude Sonnet 4",
      "model_type": "chat",
      "max_tokens": 16384,
      "context_window": 200000,
      "supports_image": true,
      "input_modalities": ["text", "image"],
      "pricing": {
        "input": 3.0,
        "output": 15.0
      },
      "available_accounts": ["1", "2", "3"]
    }
  ],
  "total": 1
}
```

### POST /api/admin/models/refresh

刷新所有账号的模型列表。

**响应示例**:

```json
{
  "success": true,
  "message": "刷新完成: 成功 3 个账号, 失败 0 个账号",
  "total_accounts": 3,
  "failed_accounts": 0
}
```

### POST /api/admin/models/refresh/:account_id

刷新指定账号的模型列表。

**响应示例**:

```json
{
  "success": true,
  "message": "账号 1 刷新成功，获取 15 个模型",
  "account_id": 1,
  "model_count": 15
}
```
